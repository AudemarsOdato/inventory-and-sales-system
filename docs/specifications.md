# Specifications

## Input specifications
- Be able to create or add new product and input their product name, price, and quantity
- Be able to record sales and update the inventory
- Must input the product, quantity, and the amount of received payment when making a sale
- Be able to update stocks or inventory when re-stocking inventory
- Be able to update product details such as name and price

## Process specifications
- Program must create a unique id for each product in inventory
- Program must also add the time and date when adding or creating new product in the inventory
- Must also add the time and date in re-stocking the inventory to know when is the last re-stocking
- Program must also add time and date when making a sale
- Calculate the change by calculating the total price first (quantity x price), then subtract it by the received paymnent amount

## Output specifications
- UI must be easy to follow and user friendly for non technical store owners e.g. buttons and descriptions
- Be able to track current level or quantity of stocks/inventory of products
